# Contributing

## Contribution focus for v0.1
- contract clarity
- CLI reliability
- docs quality
- generalized templates
- compatibility with Goose workflows

## Contribution rules
- keep templates generic and team-owned
- avoid personal details in defaults
- include verification steps in PR descriptions
- keep changes scoped to one theme per PR

## Good first contributions
- add workflow examples for additional domains
- improve validation messages
- add docs examples with expected outputs
