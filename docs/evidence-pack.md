# Evidence Pack for Quality and Bonus Scoring

Use this document to prepare review-ready evidence for ambassador submissions and partner evaluations.

## Goal
Demonstrate that Open Agent Harness is not only publishable, but measurably useful.

## Evidence categories

### 1) Project relevance evidence
- explicit links to Goose and AGENTS.md integration docs
- clear description of how contribution supports AAIF projects

### 2) Developer usefulness evidence
- quickstart path from install to first successful run
- examples showing how validation and workflow gates are used
- examples showing profile customization by a team

### 3) Quality evidence
- docs coverage map
- command reliability examples
- evidence that templates are generalized and team-owned

### 4) Outcome evidence
- before and after CI failure trend
- before and after convention adherence trend
- examples of prevented rework
- reflection artifact excerpts that produced convention upgrades

### 5) Amplification evidence
- social post links
- talk, workshop, or walkthrough links
- references to community adoption or feedback

## Reviewer-ready bundle checklist
Include these links in one submission note:
- repository root
- npm package pages
- quickstart
- CLI reference
- Goose integration page
- one pilot metrics summary
- one case study style summary

## Suggested evidence cadence
- weekly snapshot during pilot
- one monthly rollup for external sharing
- one release note per package version with quality changes
