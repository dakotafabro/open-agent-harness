# AI Collaboration Maturity Model in the Harness

## Purpose
Encode developmental growth into everyday execution so teams improve quality over time.

## Dimensions used in practice
- Intent clarity
- Context quality
- Policy discipline
- Workflow reliability
- Verification depth
- Reflection quality
- Impact efficiency

## Mechanisms
Each dimension maps to explicit mechanisms:
- intent template before generation
- context source declaration
- policy-constrained model routing
- phase and gate workflows
- validation outputs with pass or fail
- reflection artifact with promotion candidates

## Maturity progression signal
A team progresses when:
- fewer avoidable failures reach CI
- convention adherence increases
- reflection outputs become actionable and reusable
- cost per accepted outcome decreases
